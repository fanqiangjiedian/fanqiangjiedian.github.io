详细配置指南：一步步设置Shadowrocket

手动添加代理服务器

对于希望精细控制代理设置的用户，手动添加是最直接的方式：

  1. 打开Shadowrocket应用，点击底部导航栏的"配置"标签
  2. 选择"添加代理"选项

  3. 根据您的代理服务提供商提供的信息填写以下内容：

    • 类型：选择对应的协议（HTTP/HTTPS/SOCKS5/VMess等）
    • 地址：输入代理服务器的IP地址或域名
    • 端口：服务器监听的端口号
    • 用户验证（如需要）：填写用户名和密码
    • 加密方式（针对某些协议）：选择适当的加密算法

  4. 填写完成后点击"保存"，新代理将出现在您的代理列表中

导入配置文件

对于拥有复杂代理设置或订阅了多个节点的用户，导入配置文件是更高效的方式：

  1. 获取配置文件（通常为.conf或.json格式）：

    • 从您的代理服务提供商处下载
    • 或通过订阅链接自动获取

  2. 在Shadowrocket中导入配置：

    • 点击"配置"标签
    • 选择"扫描"（扫描二维码配置）或"导入"（从文件导入）
    • 按照提示完成导入过程

  3. 导入成功后，系统会自动解析配置文件并添加所有代理节点

节点选择与优化

配置完成后，选择最优节点对获得最佳体验至关重要：

  1. 节点测速：Shadowrocket内置测速功能，可以评估各节点的延迟和速度
  2. 地理位置考量：选择物理距离较近的节点通常能获得更好的速度
  3. 负载平衡：如果支持，可以设置自动切换节点以实现负载均衡
  4. 备用节点：配置备用节点以防主节点失效

高级安全设置：保护您的数字足迹

为了最大限度地保护您的隐私和安全，Shadowrocket提供了一系列高级设置选项：

  1. DNS设置：

    • 使用可信的DNS服务器（如1.1.1.1或8.8.8.8）
    • 启用DNS over HTTPS/TLS防止DNS查询被窥探

  2. 防泄漏保护：

    • 启用"防止WebRTC泄漏"选项
    • 关闭"本地中继"以减少潜在的安全风险

  3. 分流规则：

    • 设置智能路由，让特定应用或网站走代理，其他流量直连
    • 配置广告拦截规则提升浏览体验

  4. 全局与分应用代理：

    • 可选择全局代理（所有流量通过代理）
    • 或分应用代理（仅指定应用使用代理）

FAQ

Sing-box 的配置文件与 Clash 有何区别？
Sing-box 的配置使用 JSON 格式，而 Clash 采用 YAML 格式。前者更适合程序自动解析，后者更便于人工编辑。Sing-box 在性能和扩展性上有明显优势，支持更细粒度的流量控制。
Matsuri 节点排序如何提高视频播放流畅性？
Matsuri 会测速节点延迟，并结合策略组自动选择延迟低、稳定性高的节点处理视频流量。减少缓冲和卡顿，保证在线视频播放顺畅，提高整体网络体验。
Surfboard 可以导入哪些格式的规则文件？
Surfboard 支持 YAML 和部分 JSON 格式的规则文件，用户可以导入节点、策略组和分流规则。注意文件格式和语法必须符合 Surfboard 规范，否则可能导入失败。
VLESS + Reality 与 Trojan 在性能上差异大吗？
两者性能相近。VLESS + Reality 伪装更自然，适合强封锁地区；Trojan 则依托标准 TLS 实现高速连接。选择取决于服务器支持与环境稳定性。
Matsuri 的节点测速结果能影响策略组吗？
可以。Matsuri 会将延迟最低的节点自动标记为优先节点，结合策略组，程序会优先选择最佳节点处理流量，保证访问速度和连接稳定性。
Matsuri 节点延迟排序功能如何使用？
Matsuri 会自动测试节点延迟，并根据结果排序。用户可以选择延迟最低的节点使用，从而优化访问速度和稳定性，尤其适合游戏、视频和海外网站加速。
Trojan 的证书配置错误会导致什么问题？
如果 TLS 证书无效或域名不匹配，Trojan 客户端将无法建立安全连接，导致节点无法使用。需要确保证书有效、域名正确，才能保证正常访问。
Matsuri 节点延迟排序功能如何使用？
Matsuri 会定期测速节点延迟，并结合策略组自动选择延迟最低节点。用户无需手动切换即可保持高速访问，优化网页、视频和游戏体验，提高网络稳定性。
SSR 如何手动导入节点？
SSR 节点通常以“ssr://”链接形式提供。用户可在客户端中点击“添加节点”或“导入链接”，粘贴该字符串后保存即可。程序会自动解析服务器地址、端口、加密方式等参数。
Mellow 与 Clash 的核心区别是什么？
Mellow 采用基于透明代理的架构，适合路由或系统级流量重定向，而 Clash 更偏向规则式应用代理。Mellow 的灵活性更高，但配置复杂；Clash 则兼顾易用性与功能扩展性。
